# ハンドメイド作家向けアプリ エコシステム ロードマップ

## ビジネスライフサイクル分析

ハンドメイド作家のビジネスを「材料仕入れ → 制作 → 出品 → 販売 → 発送 → 経理」の流れで分析し、各フェーズのカバー状況と今後の方針を整理する。

### 現在のカバー状況

| フェーズ | 状況 | 対応アプリ | 主な機能 |
|---------|------|-----------|---------|
| 材料仕入れ | △ 部分対応 | cost-app | 材料費の記録（仕入先管理は未対応） |
| 制作 | × 未対応 | - | 作業時間の記録、進捗管理が必要 |
| 原価計算 | ◎ 対応済み | cost-app | 9種類のコストカテゴリ（材料/梱包/人件費/外注/開発/設備/物流/電気/手数料）で原価算出。シミュレーション機能付き |
| 在庫管理 | ◎ 対応済み | cost-app | 商品別の在庫数管理 |
| 出品 | ◎ 対応済み | pfauto-app | minne/Creema/BASE/iichiへの自動出品 |
| 受注 | ◎ 対応済み | shipping-manager | Gmail/Playwright で注文を自動取得 |
| 発送 | ◎ 対応済み | shipping-manager | Click Post/ヤマト伝票作成 |
| 顧客管理 | △ 部分対応 | shipping-manager | 購入者一覧・履歴（CRM機能は限定的） |
| 売上分析 | × 未対応 | - | 売上推移、商品別・PF別分析が必要 |
| 経理・税務 | × 未対応 | - | 経費記録、確定申告準備が必要 |
| 作業管理 | × 未対応 | - | 制作時間トラッキング、時給計算が必要 |

## 技術スタック

全アプリで統一している技術方針：

```
共通スタック:
  - Next.js 16 (App Router) + React 19 + TypeScript
  - TailwindCSS 4 + shadcn/ui
  - ヘキサゴナルアーキテクチャ (Ports & Adapters)
  - Vitest (テスト)
  - Vercel (デプロイ)

データストア:
  - cost-app:              Supabase (PostgreSQL) + localStorage (ゲスト) + Google Sheets
  - shipping-manager:      Supabase (PostgreSQL) + Google Sheets
  - pfauto-app:            Google Sheets（マスターデータ）
  - 領収書アプリ（予定）:   Supabase (PostgreSQL)

外部連携:
  - Google Sheets API      (全3アプリ)
  - Gmail API              (shipping-manager, pfauto-app)
  - Playwright             (shipping-manager, pfauto-app)
  - Slack Webhook          (全3アプリ)
  - Click Post API         (shipping-manager)
  - ヤマトPUDO             (shipping-manager)
  - Resend (メール送信)    (cost-app - チーム招待)
  - microCMS               (cost-app - 画像ホスティング)

既存のパッケージ化基盤（cost-appに実装済み）:
  - チーム機能（オーナー/管理者/メンバー/ビューアー）
  - RLS (Row Level Security) によるユーザー/チーム単位のデータ分離
  - 監査ログ（データ変更の記録・エクスポート）
  - Excel/PDF/CSVエクスポート（ExcelJS, jsPDF）
  - 9種類のコストカテゴリによる詳細な原価計算
  - コストシミュレーション（変動、目標利益率、設備投資）
```

## ロードマップ詳細

### Phase 1: 領収書整理アプリ（最優先）

**理由**: 確定申告は全作家に必須。手作業の経費管理は最も時間がかかる作業の一つ。

**スコープ**:
- レシートの手動登録・管理
- 確定申告の勘定科目に対応したカテゴリ分類
- インボイス制度対応
- 月次/年次レポートとCSVエクスポート

詳細: [receipt-app-spec.md](receipt-app-spec.md)

### Phase 2: 売上分析（shipping-manager拡張）

**理由**: 新規アプリ不要。既存の受注データを活用して分析機能を追加。

**スコープ**:
- 売上推移グラフ（日次/週次/月次）
- 商品別売上ランキング
- プラットフォーム別比較（minne vs Creema vs BASE vs iichi）
- 季節・曜日別の販売傾向
- リピーター率

### Phase 3: 作業時間トラッカー

**理由**: 制作時間を正確に記録することで、cost-appの原価計算（人件費）の精度が大幅に向上。

**スコープ**:
- タイマー式の作業時間記録（開始/停止）
- 作業カテゴリ（制作、撮影、梱包、事務作業）
- 商品別・カテゴリ別の時間集計
- 時給計算と原価への反映
- cost-appとの連携（人件費の自動算出）

### Phase 4: 統括ダッシュボード（本リポジトリ）

**理由**: アプリ数が増えた段階で、一画面で全体を俯瞰できるダッシュボードが必要。

**スコープ**:
- 各アプリへのリンクとステータス表示
- 横断サマリー（売上、経費、在庫、未発送件数）
- クイックアクション（よく使う操作へのショートカット）
- 通知の一元管理

### Phase 5: 手順書・マニュアル作成アプリ

**理由**: パッケージ化の前段階として、各アプリの使い方と制作手順を文書化する基盤が必要。

**スコープ**:
- Markdownベースの手順書エディタ
- 画像・動画の埋め込み
- ステップバイステップの手順テンプレート
- カテゴリ別の整理（アプリ操作、制作手順、梱包手順 etc.）
- PDF/HTML出力

### Phase 6: パッケージ化・マルチテナント対応

**理由**: 他のハンドメイド作家に提供するための基盤整備。

**スコープ**:
- 共通認証基盤（シングルサインオン）
- マルチテナント対応（ユーザー別データ分離）
- セットアップウィザード（初回導入の簡易化）
- 利用プラン管理（フリー/プレミアム）
- ドキュメントサイト

## 追加で検討すべきアプリ案

以下は現時点では優先度が低いが、将来的に検討する価値があるアプリ：

### 顧客CRM
- shipping-managerのbuyer機能を独立・拡張
- リピーター管理、購入頻度分析
- お礼メッセージ・フォローアップの自動化
- 顧客タグ付け（好みの傾向、贈り物利用 etc.）
- **ハンドメイド特有のニーズ**: リピーターとの関係構築が売上の生命線

### 素材・仕入れ管理
- 仕入先マスタ（店名、URL、連絡先）
- 素材別の単価履歴（価格変動の追跡）
- 在庫切れアラートと発注タイミング通知
- cost-appとの連携（材料費の自動更新）

### 写真・画像管理
- 商品撮影の一元管理
- PF別の画像要件に合わせたリサイズ・クロップ
- 撮影セットのテンプレート管理
- pfauto-appとの連携

### レビュー・評価モニター
- 各PF（minne, Creema, BASE, iichi）のレビューを一元収集
- 評価推移のモニタリング
- 返信テンプレート管理
- ネガティブレビューのアラート

### イベント・マルシェ管理
- 出店予定カレンダー
- 持ち込み在庫リスト（cost-app連携）
- イベント別売上記録
- 出店費用の管理（領収書アプリ連携）
- 必要備品チェックリスト

## パッケージ化に向けた共通基盤

将来のパッケージ化に備え、以下の共通モジュールを段階的に整理する。
**注**: cost-appにはチーム機能・RLS・監査ログが既に実装されており、これをベースに他アプリへ展開する方針が効率的。

| モジュール | 概要 | 対象アプリ | 備考 |
|-----------|------|-----------|------|
| 共通UIコンポーネント | shadcn/uiベースの共通テーマ・コンポーネント | 全アプリ | |
| 認証基盤 | Supabase Auth（共通ログイン・ユーザー管理） | 全アプリ | cost-app, shipping-managerで実装済み |
| チーム・マルチテナント | ロール管理、RLS、データ分離 | 全アプリ | **cost-appに実装済み** → 他アプリへ展開 |
| 監査ログ | データ変更の記録・エクスポート | 全アプリ | **cost-appに実装済み** → 他アプリへ展開 |
| 通知基盤 | Slack・メール通知の統一インターフェース | 全アプリ | 全アプリで個別実装済み → 共通化 |
| Google Sheets連携 | 共通のSpreadsheet読み書きユーティリティ | 全3アプリ | 全アプリで個別実装済み → 共通化 |
| Playwright基盤 | ブラウザ自動化の共通フレームワーク | pfauto-app, shipping-manager | |
| エクスポート基盤 | Excel/PDF/CSV出力の共通ユーティリティ | 全アプリ | **cost-appにExcelJS/jsPDF実装済み** |
